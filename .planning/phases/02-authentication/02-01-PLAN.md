<task type="auto">
  <name>Task 2: Replace middleware with clerkMiddleware</name>
  <files>
    - apps/web/middleware.ts
  </files>
  <action>
Replace the existing custom CSP middleware with Clerk's clerkMiddleware that handles both authentication AND CSP.

The new middleware should:
1. Import `clerkMiddleware` and `createRouteMatcher` from `@clerk/nextjs/server`
2. Define public routes: `/`, `/auth(.*)`, `/api/webhooks(.*)`
3. Define protected routes: `/dashboard(.*)`, `/settings(.*)`, `/chart(.*)`
4. Use `contentSecurityPolicy: { strict: true }` for automatic nonce handling
5. Add custom CSP directives for images (blob:, data:, https:) and fonts (fonts.gstatic.com)
6. Keep the same matcher config pattern from existing middleware

Key differences from Phase 1 middleware:
- Clerk handles nonce generation automatically
- No manual x-nonce header needed
- Other security headers (X-Frame-Options, etc.) should be added in Next.js config or via response headers in middleware

Preserve existing security headers (X-Frame-Options: DENY, X-Content-Type-Options: nosniff, etc.) by setting them on the response after clerkMiddleware processes.

Reference the research document Pattern 2 for the exact implementation.
