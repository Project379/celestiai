1. Create `apps/web/lib/stripe/subscription.ts` — subscription lifecycle handler functions:

   Import `stripe` from `@/lib/stripe/client`, import `createServiceSupabaseClient` from `@/lib/supabase/service`.

   **handleCheckoutComplete(session: Stripe.Checkout.Session):**
   - Extract `clerkUserId` from `session.metadata`. Throw if missing.
   - Retrieve full subscription via `stripe.subscriptions.retrieve(session.subscription as string)`.
   - Update users table (service client, where `clerk_id = clerkUserId`):
     - `subscription_tier: 'premium'`
     - `stripe_customer_id: session.customer as string`
     - `stripe_subscription_id: subscription.id`
     - `subscription_expires_at: new Date(subscription.current_period_end * 1000).toISOString()` (Stripe timestamps are Unix SECONDS — multiply by 1000 for JS Date)
     - `updated_at: new Date().toISOString()`

   **handleSubscriptionUpdated(sub: Stripe.Subscription):**
   - Extract `clerkUserId` from `sub.metadata`. Throw if missing.
   - Determine tier: `sub.status === 'active' || sub.status === 'trialing' ? 'premium' : 'free'`.
   - Update users: `subscription_tier`, `stripe_subscription_id`, `subscription_expires_at`, `updated_at`.

   **handleSubscriptionDeleted(sub: Stripe.Subscription):**
   - Extract `clerkUserId` from `sub.metadata`. Throw if missing.
   - Update users: `subscription_tier: 'free'`, `stripe_subscription_id: null`, `subscription_expires_at: null`, `updated_at`.

   **handleInvoicePaid(invoice: Stripe.Invoice):**
   - If `!invoice.subscription` return early.
   - Retrieve subscription via `stripe.subscriptions.retrieve(invoice.subscription as string)`.
   - Extract `clerkUserId` from `sub.metadata`. If missing, return (don't throw — invoice events can be noisy).
   - Update users: `subscription_tier: 'premium'`, `subscription_expires_at` refreshed, `updated_at`.

   Export all four functions.

2. Create `apps/web/app/api/webhooks/stripe/route.ts` — POST handler:

   **CRITICAL: Use `await request.text()` NOT `request.json()`. No bodyParser config export needed in App Router.**

   - Read raw body: `const body = await request.text()`
   - Get signature: `const sig = request.headers.get('stripe-signature')!`
   - Verify signature: `stripe.webhooks.constructEvent(body, sig, process.env.STRIPE_WEBHOOK_SECRET!)` — wrap in try/catch, return 400 on failure with log.
   - Create service Supabase client.
   - **Idempotency check:** Query `processed_webhook_events` where `stripe_event_id = event.id`. If found, return 200 immediately ("OK").
   - **Event switch:**
     - `checkout.session.completed`: Cast `event.data.object` as `Stripe.Checkout.Session`. Only process if `session.mode === 'subscription'`. Call `handleCheckoutComplete`.
     - `customer.subscription.updated`: Cast as `Stripe.Subscription`. Call `handleSubscriptionUpdated`.
     - `customer.subscription.deleted`: Cast as `Stripe.Subscription`. Call `handleSubscriptionDeleted`.
     - `invoice.paid`: Cast as `Stripe.Invoice`. Call `handleInvoicePaid`.
     - `invoice.payment_failed`: Log warning only (`console.warn`). No action — Stripe retries automatically.
   - **After successful processing:** Insert into `processed_webhook_events` table: `stripe_event_id`, `event_type`, `processed_at`.
   - **On processing error:** Return 500 (NOT 200) so Stripe retries. Log the error with event.id.
   - **On success:** Return 200.
  </action>

   - State: `activated` (boolean), initialized to `initialTier === 'premium'`.
   - **Polling approach** (simpler and more reliable than Supabase Realtime which may need publication configuration):

     - Actually, to keep it simpler: create a small helper API or use the existing user endpoint. **Preferred approach:** Poll the `/api/user` endpoint or create a minimal `/api/stripe/status` endpoint.
