1. Create `apps/web/app/api/stripe/cancel/route.ts`:

   **POST — Cancel subscription (set cancel_at_period_end: true):**
   - Auth: `const { userId } = await auth()`. Return 401 with `{ error: "Неоторизиран достъп" }`.
   - Look up user's `stripe_subscription_id` from Supabase users table (service client, `clerk_id = userId`).
   - If no subscription ID, return 400 `{ error: "Няма активен абонамент" }`.
   - Call `stripe.subscriptions.update(subscriptionId, { cancel_at_period_end: true })`.
   - Optionally accept `{ reason: string }` from request body for product feedback logging (console.log or store — at Claude's discretion).
   - Return `{ success: true }`.

   **DELETE — Reactivate subscription (set cancel_at_period_end: false):**
   - Auth check same as above.
   - Look up `stripe_subscription_id`.
   - Call `stripe.subscriptions.update(subscriptionId, { cancel_at_period_end: false })`.
   - Return `{ success: true }`.

2. Create `apps/web/app/api/stripe/portal/route.ts`:

   **POST — Create Stripe Customer Portal session:**
   - Auth check.
   - Look up user's `stripe_customer_id`. If none, return 400 `{ error: "Няма свързан акаунт за плащане" }`.
   - Create portal session: `stripe.billingPortal.sessions.create({ customer: stripeCustomerId, return_url: ${NEXT_PUBLIC_APP_URL}/settings })`.
   - Return `{ url: portalSession.url }`.

3. Create `apps/web/app/(protected)/settings/page.tsx`:

   **Server component wrapper:**
   - Auth check, get userId from Clerk.
   - Fetch user from Supabase users table: `subscription_tier`, `stripe_subscription_id`, `subscription_expires_at`, `stripe_customer_id`.
   - If user has `stripe_subscription_id`, fetch subscription details from Stripe: `stripe.subscriptions.retrieve(subscriptionId, { expand: ['default_payment_method'] })`.
   - Extract: `status`, `cancel_at_period_end`, `current_period_end`, `default_payment_method` (last4, brand).
   - Also extract the price to determine plan name (monthly vs annual).
   - Pass all data to client component `SettingsContent`.

   **Client component `SettingsContent`:**
   - Props: `tier`, `subscriptionData` (nullable — free users have none).

   **Layout:** Page title "Настройки" (Settings). Subscription section as a glassmorphic card.

   **States:**

   a) **Free user (no subscription):**
      - Show: "Безплатен план" (Free plan).
      - Brief description of what premium unlocks.
      - "Отключи Премиум" button linking to `/pricing`.

   b) **Active subscription (not cancelling):**
      - Plan name: "Celestia Премиум" + "(Месечен)" or "(Годишен)" based on price interval.
      - Status badge: "Активен" (Active) with green accent.
      - Next billing date: formatted as Bulgarian locale date from `current_period_end * 1000`.
      - Payment method: "[Brand] •••• [last4]" (e.g., "Visa •••• 4242").
      - "Управление на плащанията" (Manage payments) button — calls `/api/stripe/portal`, redirects to `portalSession.url`.
      - "Отказ от абонамент" (Cancel subscription) button — opens confirmation dialog.

   c) **Cancelling (cancel_at_period_end: true):**
      - Status badge: "Отменен" (Cancelling) with amber/yellow accent.
      - "Премиум достъпът ви изтича на [date]" (Your premium expires on [date]).
      - "Възстанови абонамент" (Reactivate) button — calls `DELETE /api/stripe/cancel` to set `cancel_at_period_end: false`. On success, refresh page.
      - Still show billing history link.

   d) **Expired (tier is free but had subscription before — subscription_expires_at in past):**
      - Show: "Абонаментът ви е изтекъл" (Your subscription has expired).
      - "Абонирай се отново" button linking to `/pricing`.

   **Cancellation dialog** (use native `<dialog>` element, per Phase 2 pattern):
   - Title: "Сигурни ли сте, че искате да се откажете?"
   - Body: "Достъпът ви до премиум функциите ще продължи до [end_date]."
   - Optional "Защо се отказвате?" dropdown with options:
     - "Твърде скъпо" (Too expensive)
     - "Не използвам достатъчно" (Don't use enough)
     - "Не отговаря на очакванията ми" (Doesn't meet expectations)
     - "Друга причина" (Other reason)
   - "Потвърди отказ" (Confirm cancel) button — calls `POST /api/stripe/cancel` with optional reason, then refreshes.
   - "Запази абонамент" (Keep subscription) button — closes dialog.
   - Loading state on confirm button while API call in progress.
