  <action>
    Install dependencies in apps/web:
    ```bash
    cd apps/web && pnpm add ai @ai-sdk/google @ai-sdk/react
    ```

    Create `apps/web/app/api/oracle/generate/route.ts`:
    - `export const maxDuration = 60` at top (prevent Vercel timeout)
    - POST handler with these steps:
      1. **Auth check:** `const { userId } = await auth()` — return 401 JSON if not authenticated (Bulgarian: "Неоторизиран достъп")
      2. **Parse body:** `const { chartId, topic, regenerate } = await req.json()` — validate topic is one of 'general' | 'love' | 'career' | 'health'
      3. **Subscription tier check:** Query users table via service role client by clerk_id = userId. If user row doesn't exist, create one with default 'free' tier (upsert pattern). If topic is love/career/health and tier is not 'premium', return 403 JSON: `{ error: 'Premium required', code: 'PREMIUM_REQUIRED' }`
      4. **Chart ownership:** Fetch chart from charts table, verify user_id === userId. Return 404 if not found, 403 if not owned.
      5. **Cache check:** Query ai_readings for (chart_id, topic) where expires_at > NOW(). If found and NOT regenerate flag, return JSON `{ content, cached: true, generatedAt }` (non-streaming cached response).
      6. **Regeneration rate limit:** If regenerate flag is true and existing reading has last_regenerated_at within 24 hours, return 429 JSON: `{ error: 'Можете да регенерирате веднъж на ден' }`
      7. **Load chart calculation:** Fetch from chart_calculations by chart_id. Return 404 if not calculated yet.
      8. **Build prompts:** `buildSystemPrompt(topic)` + `chartToPromptText(chartData)` — import from lib/oracle/
      9. **Stream:** Use `streamText` from 'ai' with `google('gemini-2.5-flash')`, temperature 0.85, maxTokens 2000
      10. **onFinish callback:** In streamText options, add `onFinish: async ({ text }) => { ... }` that:
          - Strips sentinels from text via stripSentinels()
          - Upserts into ai_readings: INSERT ON CONFLICT (chart_id, topic) DO UPDATE — set content, generated_at = now, expires_at = now + 7 days, last_regenerated_at = regenerate ? now : null, model_version = 'gemini-2.5-flash'
      11. **Return:** `result.toDataStreamResponse()` — NOT awaiting the stream, return Response immediately

    Environment variable: `GOOGLE_GENERATIVE_AI_API_KEY` — the @ai-sdk/google provider reads this automatically (no manual config needed).
  </action>
  <verify>

    2. `pnpm list ai @ai-sdk/google @ai-sdk/react --filter=web` shows packages installed

  </verify>
  <done>
    POST /api/oracle/generate route exists with auth, tier gating, cache check, rate limiting, Gemini streaming via Vercel AI SDK, and onFinish DB write. Returns toDataStreamResponse() for proper streaming.
  </done>

  <action>
    Create `apps/web/app/api/oracle/readings/route.ts`:
    - GET handler: auth check, parse chartId from URL search params
    - Query ai_readings via service role client for chart_id where expires_at > NOW()
    - Return JSON array of readings: `{ topic, content, generatedAt, expiresAt, teaserContent }`
    - If no readings found, return empty array (not 404)

    Create `apps/web/app/api/oracle/teaser/route.ts`:
    - POST handler: auth check, parse { chartId, topic } from body
    - Verify chart ownership
    - Check if teaser_content already exists for this chart_id + topic — if so, return cached teaser
    - If not, use `generateText` (non-streaming) from 'ai' with google('gemini-2.5-flash'):
      - Short prompt: "Write 2-3 sentences in Bulgarian as a mystical astrological teaser about {topic} for this chart. Be intriguing but don't reveal specifics."
      - Include chart data summary
      - temperature 0.9, maxTokens 200
    - Save teaser_content to ai_readings row (upsert — create row if needed with empty content, set teaser_content)
    - Return JSON: `{ teaserContent }`

    Create `apps/web/hooks/useOracleReading.ts`:
    - 'use client' directive
    - `useOracleReading(chartId: string)` hook that:
      - Uses `useCompletion` from '@ai-sdk/react' with api: '/api/oracle/generate', streamProtocol: 'data'
      - Exposes: completion (streaming text), isLoading, error, stop
      - `generateReading(topic: string, regenerate?: boolean)` — calls `complete('', { body: { chartId, topic, regenerate } })`
      - Manages saved readings state: fetches from /api/oracle/readings?chartId=X on mount and after generation completes
      - `savedReadings` state: Record<string, { content, generatedAt, expiresAt, teaserContent }>
      - `activeTopic` state tracking which topic is currently being generated/viewed
      - `fetchSavedReadings()` function that GETs /api/oracle/readings and updates state
      - Auto-refresh saved readings when `isLoading` transitions from true to false (generation complete)
  </action>
