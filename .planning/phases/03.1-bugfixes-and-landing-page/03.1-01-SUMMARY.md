## Decisions Made

- **auth() vs auth.protect():** Used auth() with manual userId check because auth.protect() throws redirects which return HTML. Client-side fetch expects JSON, causing "Unexpected token '<'" errors.
- **Consistent error message:** All API routes use Bulgarian "Неоторизиран достъп" (Unauthorized access) for i18n consistency.
