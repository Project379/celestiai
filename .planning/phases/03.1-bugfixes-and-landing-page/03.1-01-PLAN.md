  <action>
The issue: `auth.protect()` throws a redirect when not authenticated, which returns HTML. Client code expects JSON, causing "Unexpected token '<'" error.

**Solution:** Replace `auth.protect()` with `auth()` and manually check for userId, returning a JSON 401 error if not authenticated.

For each API route file, replace this pattern:
```typescript
// OLD (problematic)
const session = await auth.protect()
const userId = session.userId
```

With this pattern:
```typescript
// NEW (returns JSON error)
const { userId } = await auth()
if (!userId) {
  return Response.json(
    { error: 'Неоторизиран достъп' },
    { status: 401 }
  )
}
```

Apply this change to:
1. `apps/web/app/api/cities/search/route.ts` - GET handler
2. `apps/web/app/api/birth-data/route.ts` - GET and POST handlers
3. `apps/web/app/api/birth-data/[id]/route.ts` - GET, PUT, DELETE handlers

Keep the import as `import { auth } from '@clerk/nextjs/server'` (no change needed).
  </action>

  </verify>
