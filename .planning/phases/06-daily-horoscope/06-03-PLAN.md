<objective>
Implement Web Push notifications: service worker, subscribe/unsubscribe API, cron job for morning delivery, and notification opt-in UI on the dashboard.

Purpose: Delivers DAILY-04 (morning push notification) by wiring up the complete push notification pipeline — from browser subscription through server-side daily dispatch. The notification is a generic "your horoscope awaits" prompt that links to the dashboard, where personalized content is generated on visit (intentionally scoped down from personalized preview to avoid batch AI generation at cron time).

Output: Users can opt in to push notifications and receive a morning "your horoscope is ready" notification via Vercel cron at 06:00 UTC.
</objective>

  <files>
    packages/db/src/schema/push-subscriptions.ts

  </files>
  <action>
    1. Install web-push: Run `npm install web-push` and `npm install --save-dev @types/web-push` from the workspace root (or apps/web — check where dependencies go in this monorepo; follow the existing pattern for server-only deps like `ai`).

    2. Create `packages/db/src/schema/push-subscriptions.ts`:
    - Table `push_subscriptions`: `id` (uuid, defaultRandom, primaryKey), `user_id` (text, notNull), `endpoint` (text, notNull, unique), `p256dh` (text, notNull), `auth` (text, notNull), `created_at` (timestamp with timezone, defaultNow, notNull).
    - Add index on `user_id`.
    - Export table and types `PushSubscription`, `NewPushSubscription`.

    3. Update `packages/db/src/schema/index.ts` to re-export pushSubscriptions and its types. (Note: 06-01 already added dailyTransits and dailyHoroscopes — append pushSubscriptions to the same file.)

    4. Create `apps/web/public/sw.js`:
    - `push` event listener: parse `event.data.json()`, call `self.registration.showNotification(data.title, { body: data.body, icon: data.icon || '/icon-192x192.png', badge: '/badge.png', data: { url: data.url || '/dashboard' } })`.
    - `notificationclick` event listener: close notification, `clients.openWindow(event.notification.data.url)`.
    - This is a plain JS file (not TypeScript) since service workers run outside the app bundle.

    5. Create `apps/web/app/api/push/subscribe/route.ts`:
    - POST endpoint. Auth check via `auth()`. Parse body: `{ subscription: { endpoint, keys: { p256dh, auth } } }`.
    - Upsert into `push_subscriptions` with `onConflict: 'endpoint'` (same browser re-subscribing updates the row).
    - Return `{ success: true }`.

    6. Create `apps/web/app/api/push/unsubscribe/route.ts`:
    - POST endpoint. Auth check. Parse body: `{ endpoint }`.
    - Delete from `push_subscriptions` WHERE endpoint = endpoint AND user_id = userId.
    - Return `{ success: true }`.

    7. Create `apps/web/app/api/cron/daily-horoscope/route.ts`:
    - GET endpoint. Check `Authorization: Bearer ${CRON_SECRET}` header — return 401 if mismatch.
    - Import `webpush` from `web-push`. Call `webpush.setVapidDetails('mailto:hello@celestia.app', process.env.NEXT_PUBLIC_VAPID_PUBLIC_KEY!, process.env.VAPID_PRIVATE_KEY!)`.
    - Fetch all rows from `push_subscriptions`.
    - For each subscription, `webpush.sendNotification(sub, JSON.stringify({ title: 'Вашият дневен хороскоп', body: 'Новото ви послание от звездите ви очаква.', icon: '/icon-192x192.png', url: '/dashboard' }))`.
    - Wrap each send in try/catch. On 410 or 404 status code errors, delete the subscription from DB (expired endpoint cleanup).
    - Track `sent` and `failed` counts. Return `{ sent, failed }`.
    - Set `export const maxDuration = 300` (5 minutes for batch processing).
    - NOTE: The notification body is intentionally generic (not a personalized horoscope preview). Personalized content is generated when the user visits /dashboard. This avoids N concurrent AI calls during the cron job.

    8. Create `apps/web/vercel.json`:
    - `{ "$schema": "https://openapi.vercel.sh/vercel.json", "crons": [{ "path": "/api/cron/daily-horoscope", "schedule": "0 6 * * *" }] }`
    - Check if a vercel.json already exists — if so, merge the crons array into it rather than overwriting.
  </action>

    - `curl -X POST http://localhost:3000/api/push/subscribe` returns 401 without auth
    - `curl http://localhost:3000/api/cron/daily-horoscope` returns 401 without CRON_SECRET
  </verify>

    - Service worker handles push events and notification clicks
    - Subscribe/unsubscribe API routes save and remove push subscriptions
    - Cron route sends generic morning notification to all subscribers with expired endpoint cleanup
    - vercel.json configures daily 06:00 UTC cron schedule
  </done>

    - Check current permission: `Notification.permission` — 'granted', 'denied', or 'default'.
    - State: `isSubscribed` (boolean), `isLoading` (boolean).

    - Subscribe flow: `registration.pushManager.subscribe({ userVisibleOnly: true, applicationServerKey: urlBase64ToUint8Array(process.env.NEXT_PUBLIC_VAPID_PUBLIC_KEY!) })`, then POST the subscription object to `/api/push/subscribe`.
    - Unsubscribe flow: `subscription.unsubscribe()`, then POST `{ endpoint }` to `/api/push/unsubscribe`.
    - Include `urlBase64ToUint8Array` helper function (standard conversion for applicationServerKey).
    - UI: Compact banner/card with bell icon. If not subscribed: "Получавайте сутрешен хороскоп" (Receive morning horoscope) with "Включи" (Enable) button. If subscribed: "Сутрешните известия са включени" (Morning notifications enabled) with "Изключи" (Disable) button. If denied: "Известията са блокирани в браузъра" (Notifications blocked in browser) — no button.
    - Style with glassmorphism card matching dashboard theme (bg-white/5, border-white/10, etc.).
