       - `charts.ts`: Use `pgTable.withRLS()` for charts table with fields:
         - id (uuid, primary key, defaultRandom)
         - userId (text, notNull, default sql`auth.jwt()->>'sub'`)
         - name (text, notNull) - chart label like "Moiata karta"

         - birthTimeKnown (boolean, notNull, default true)
         - birthTime (text, nullable) - HH:MM format
         - approximateTimeRange (text, nullable) - 'morning'|'afternoon'|'evening'|'night'
         - cityId (uuid, nullable, references cities.id)
         - cityName (text, notNull)
         - latitude (real, notNull)
         - longitude (real, notNull)
         - createdAt/updatedAt timestamps

       - Export types: `Chart` (select) and `NewChart` (insert)

       - `cities.ts`: Regular table (no RLS - public read):
         - id (uuid, primary key)
         - name (text) - Bulgarian name
         - nameAscii (text) - ASCII for search
         - oblast (text) - region
         - ekatte (text, nullable) - national code
         - type (text) - 'city'|'town'|'village'
         - latitude/longitude (real)
         - population (real, nullable)
       - Add indexes on nameAscii and type

  <action>
    Create Supabase client factories following the 2025 Clerk native integration pattern:

    1. **packages/db/src/client.ts**: Export a generic factory that takes accessToken function:
       ```typescript
       export function createSupabaseClient(accessToken: () => Promise<string | null>) {
         return createClient(
           process.env.NEXT_PUBLIC_SUPABASE_URL!,
           process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!,
           { accessToken }
         )
       }
       ```

    2. **apps/web/lib/supabase/server.ts**: Server-side client using `auth()` from Clerk:
       ```typescript
       import { auth } from '@clerk/nextjs/server'
       export async function createServerSupabaseClient() {
         return createClient(url, key, {
           async accessToken() {
             return (await auth()).getToken()
           }
         })
       }
       ```

    3. **apps/web/lib/supabase/client.ts**: Client-side hook using `useSession`:
       ```typescript
       'use client'
       import { useSession } from '@clerk/nextjs'
       export function useSupabaseClient() {
         const { session } = useSession()
         return useMemo(() => createClient(url, key, {
           async accessToken() {
             return session?.getToken() ?? null
           }
         }), [session])
       }
       ```

    4. Add `@supabase/supabase-js` to apps/web dependencies.

    5. Update `.env.example` with placeholder Supabase variables:
       - NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
       - NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key
       - DATABASE_URL=postgresql://postgres:password@db.your-project.supabase.co:5432/postgres
