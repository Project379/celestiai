Create the API endpoint with validation and caching:

1. Add @celestia/astrology dependency to apps/web/package.json:
   - "@celestia/astrology": "workspace:*"
   - Run pnpm install

2. apps/web/lib/validators/chart.ts:
   - Zod schema for calculation input:
     ```typescript
     export const chartCalculationSchema = z.object({
       chartId: z.string().uuid({ message: 'Невалиден идентификатор' }),
     })
     ```
   - We only need chartId - birth data comes from charts table

3. apps/web/app/api/chart/calculate/route.ts:
   - POST handler following existing pattern (see birth-data/route.ts)

   Flow:
   a. Check auth() - return 401 JSON if not authenticated (Bulgarian: "Неоторизиран достъп")
   b. Validate request body with chartCalculationSchema
   c. Fetch the chart from 'charts' table by chartId
      - Verify chart.user_id matches authenticated user (security)
      - Return 404 if not found: "Картата не е намерена"
   d. Check chart_calculations for existing cached result:
      - SELECT from chart_calculations WHERE chart_id = chartId
      - If exists, return cached data immediately
   e. If no cache, calculate:
      - Import calculateNatalChart from '@celestia/astrology'
      - Call with chart.birthDate, chart.birthTime, chart.latitude, chart.longitude
      - Use noon (12:00) if birthTime is null
   f. Cache the result:
      - INSERT into chart_calculations
   g. Return ChartData JSON

   Error handling:
   - Calculation failures: 500 with "Грешка при изчисление. Моля, проверете данните."
   - Unknown chart: 404 with "Картата не е намерена"
   - Validation errors: 400 with field-specific messages
  </action>
